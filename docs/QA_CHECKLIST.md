# Чеклист приёмки после установки или обновления

Этот документ используется после штатной установки или обновления АИС УЗМО.
Он не заменяет инструкции `DEPLOY_LINUX.md` и `MAINTENANCE.md` и не содержит
команд разработки.

Для контрольных операций используйте специально созданные записи и файлы без
реальных персональных данных. После приёмки удалите их штатными средствами
приложения.

## 1. Проверка на сервере

Ответственный Linux-администратор выполняет от `root` только готовую проверку:

```bash
bash /srv/ais_uzmo/deploy/check.sh
```

Последняя строка должна быть:

```text
Все проверки пройдены.
```

Скрипт проверяет службы, миграции, PostgreSQL, Redis, локальные служебные порты,
Nginx, статические файлы, закрытый каталог `media`, постоянные настройки панели,
резервную копию и права на секретные настройки. Не заменяйте его отдельными
командами `manage.py`, не запускайте на сервере проверки исходного кода и не
создавайте миграции.

Если проверка завершилась ошибкой, остановите приёмку, сохраните её вывод и
действуйте по [`MAINTENANCE.md`](MAINTENANCE.md). Не удаляйте `.env`, базу,
`media`, `runtime` или резервные копии и не запускайте установщик повторно.

## 2. Вход и навигация

- Страница входа открывается по адресу, указанному при установке.
- Первый администратор входит в систему.
- Главный рабочий экран открывается без ошибок.
- Раздел `/control/` доступен только пользователю с соответствующими правами.
- Стандартный раздел `/admin/` доступен только уполномоченному пользователю.
- Выход из системы завершает сеанс.

## 3. Заявки

- Выбор территориального органа, отдела и таблицы работает.
- Контрольная заявка создаётся и открывается.
- Редактирование и изменение статуса сохраняются.
- История статусов отображается корректно.
- Повтор номера в том же территориальном органе и отделе отклоняется понятным
  сообщением.
- Одинаковый номер в разных органах или отделах обрабатывается согласно
  установленному бизнес-правилу.
- В колонке «Номер заявки / Исходящий» номер заявки расположен сверху, а
  последний ответ — ниже тонкой разделительной линии.
- К одной заявке добавляются два ответа без ручного выбора типа; в карточке
  самый новый помечен как «Итоговый», предыдущий —
  «Промежуточный», а в таблице видны последний ответ и количество остальных.
- Добавление, изменение и удаление ответа не меняет статус заявки и не создаёт
  запись в истории статусов.
- Повтор номера ответа в пределах одной заявки отклоняется; такой же номер ответа
  у другой заявки допускается.
- Поиск находит заявку по номеру и примечанию любого её ответа без дублирования
  строки заявки.
- CSV- и Excel-выгрузки содержат номер заявки и все относящиеся к ней ответы.
- Пользователь без права записи может просматривать ответы, но не может добавить,
  изменить или удалить их, в том числе по прямой ссылке.
- Операции с ответами отображаются в журнале как действия с соответствующей
  заявкой.
- Удаление и восстановление выполняются только пользователем с нужными правами.

## 4. ТМЦ

- Контрольная ТМЦ-заявка создаётся.
- Поиск и выбор ТМЦ работают.
- Проверки количества и наличия срабатывают.
- Ошибки формы выводятся понятным текстом.

## 5. Фотографии

- Папка фотографий создаётся.
- Одна и несколько контрольных фотографий загружаются.
- Превью, lightbox и описание работают.
- Фотография прикрепляется к заявке.
- Скачивание отдельной фотографии, папки и архива работает.
- Файл, который не является допустимым изображением, отклоняется.
- Фотография открывается только через разрешённые действия приложения; прямой
  доступ к файлам из каталога `media` не предоставляется.
- Пользователь без доступа к соответствующим данным не может открыть фотографию
  по прямой ссылке.

## 6. Административные разделы

- Оперативная сводка и графики открываются.
- Фильтры по датам, органам, отделам, статусам и поиску работают.
- Разделы «Заявки», «Территориальные органы», «Отделы» и «Материальная база»
  показывают согласованные данные.
- Список сотрудников и статусы присутствия отображаются.
- Создание сотрудника и изменение его прав работают.
- Блокировка, разблокировка и сброс активации работают согласно полномочиям.
- Настройки порогов сохраняются.

## 7. Журнал действий

- Журнал открывается только пользователю с нужными правами.
- Фильтры по пользователю, действию, объекту и дате работают.
- Детальная карточка показывает понятные значения «было / стало».
- Создание, изменение и удаление контрольных данных отражаются в журнале.

## 8. Права доступа

Проверить подготовленными учётными записями разных ролей:

- руководитель имеет предусмотренный полный доступ;
- администратор видит только разрешённые административные разделы;
- оператор видит только назначенные органы и отделы;
- наблюдатель не может изменять данные;
- обычный сотрудник без административных прав не открывает `/control/`;
- прямой URL не позволяет получить чужие данные или обойти проверку роли.

## 9. Интерфейс

- Стили, значки и графики загружаются.
- Модальные окна открываются и закрываются.
- Динамические действия выполняются без полной перезагрузки там, где это
  предусмотрено.
- Ошибки запросов не остаются без сообщения пользователю.
- Основные страницы остаются пригодными для работы при поддерживаемом масштабе
  браузера и разрешении экрана.

## 10. Завершение приёмки

- Контрольные заявки, сотрудники, папки и фотографии удалены или оформлены в
  соответствии с согласованным порядком.
- Результат `deploy/check.sh` и номер принятой версии зафиксированы.
- Обнаруженные отклонения записаны с адресом страницы, ролью пользователя,
  временем и последовательностью действий.
- Приёмка подтверждена ответственным сотрудником.
