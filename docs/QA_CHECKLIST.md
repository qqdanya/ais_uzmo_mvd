# Чеклист проверки после установки

Чеклист используется после локального запуска, тестового развёртывания или обновления сервера.

## 1. Автоматические проверки

```bash
python manage.py check
python manage.py makemigrations --check --dry-run
python manage.py test --parallel 4
python scripts/refactor_static_check.py
```

Для production-настроек:

```bash
python manage.py check --deploy --settings=config.settings_prod
python manage.py makemigrations --check --dry-run --settings=config.settings_prod
```

## 2. Вход и базовая навигация

- Страница входа открывается.
- Руководитель входит в систему.
- Главный рабочий экран открывается без ошибок.
- Административная панель `/control/` открывается для пользователя с правами.
- Стандартная Django admin-панель `/admin/` доступна только уполномоченному пользователю.

## 3. Заявки

- Выбор территориального органа работает.
- Переключение отделов и таблиц работает.
- Создание заявки работает.
- Редактирование заявки работает.
- Удаление или soft-delete работает ожидаемо.
- Изменение статуса заявки сохраняется.
- История статусов отображается.
- Нельзя создать дубль номера заявки внутри одного территориального органа и отдела.
- Можно использовать одинаковый номер в разных органах или разных отделах, если это соответствует бизнес-правилу.

## 4. ТМЦ

- Добавление ТМЦ-заявки работает.
- Подбор ТМЦ через поиск/autocomplete работает.
- Проверки количества и наличия срабатывают.
- Ошибки формы отображаются понятным текстом.

## 5. Фотографии

- Создание папки работает.
- Загрузка одной фотографии работает.
- Массовая загрузка фотографий работает.
- Просмотр через lightbox работает.
- Редактирование описания фотографии работает.
- Прикрепление фотографии к заявке работает.
- Скачивание отдельной фотографии, папки или архива работает.
- Файл с неподходящим содержимым не принимается как изображение.

## 6. Административная панель

- Оперативная сводка открывается.
- Фильтры по датам и территориальным органам работают.
- Раздел «Заявки» фильтрует по органам, отделам, статусам и поиску.
- Раздел «Территориальные органы» показывает корректные показатели.
- Раздел «Отделы» показывает корректные показатели.
- Раздел «Материальная база» показывает актуальные сведения.
- Раздел «Сотрудники» показывает список и онлайн-статусы.
- Создание сотрудника работает.
- Редактирование прав сотрудника работает.
- Блокировка, разблокировка и сброс активации работают.
- Настройки порогов сохраняются.

## 7. Журнал действий

- Журнал действий открывается.
- Фильтры по пользователю, действию, объекту и дате работают.
- Детальная карточка действия открывается.
- Изменения «было / стало» отображаются понятно.
- Пользователь без нужных прав не получает доступ к журналу.

## 8. Права доступа

- Руководитель имеет полный доступ.
- Администратор видит только разрешённые разделы.
- Оператор видит только доступные территориальные органы и отделы.
- Наблюдатель не может изменять данные.
- Обычный сотрудник без административных прав не открывает `/control/`.
- Пользователь не может получить доступ к чужим данным через прямой URL.

## 9. Static и интерфейс

- CSS загружается.
- Bootstrap-модальные окна открываются.
- Bootstrap Icons отображаются.
- HTMX-запросы работают.
- График оперативной сводки отображается.
- Ошибки AJAX/HTMX не остаются без сообщения пользователю.

## 10. Файлы, которые не должны попасть в релиз

Перед передачей или коммитом проверить, что отсутствуют:

```text

db.sqlite3
*.sqlite3
media/
staticfiles/
__pycache__/
*.pyc
.venv/
venv/
.pytest_cache/
.coverage
htmlcov/
*.log
*.zip
dashboard_thresholds.json
dashboard_thresholds.json.tmp
.idea/
.vscode/
```
