# Установка АИС УЗМО на Ubuntu

Эта инструкция предназначена для сотрудника ИЦ и описывает **чистую установку**
на отдельный сервер Ubuntu 24.04.

Здесь приведён быстрый штатный вариант через готовый установщик. Если
администратору нужно выполнить и проверить каждый этап самостоятельно, используйте
отдельную [`инструкцию по ручному развёртыванию`](DEPLOY_LINUX_MANUAL.md). Два
варианта нельзя смешивать или последовательно запускать на одном сервере.

Если на сервере уже работает старая версия АИС УЗМО или нужно перенести готовую
базу и фотографии, не начинайте чистую установку. Сначала запросите у разработчика
комплект переноса данных.

## 1. Что потребуется

- сервер Ubuntu 24.04 с доступом `root` или правом `sudo`;
- IP-адрес или внутреннее имя сервера;
- адрес доверенной сети, из которой разрешён вход в систему;
- три файла от разработчика:

```text
ais_uzmo-1.0.0.tar.gz
SHA256SUMS
RELEASE.txt
```

Номер версии в имени может отличаться.

Серверу нужен доступ к репозиториям пакетов Ubuntu и Python: напрямую либо через
внутренние зеркала ИЦ. Если нет ни Интернета, ни внутренних зеркал, эта инструкция
не подходит: разработчик должен заранее подготовить полный офлайн-комплект именно
для версии Ubuntu и архитектуры сервера.

## 2. Передача файлов на сервер

Передавайте файлы только способом, который разрешён ответственным за
информационную безопасность: через защищённый внутренний обмен или учтённый
носитель.

Значение SHA-256 должно быть подтверждено по согласованному независимому каналу
или электронной подписью. Получение архива и контрольной суммы из одного
непроверенного источника не подтверждает автора комплекта.

Если разрешено копирование по SSH, на рабочем компьютере можно выполнить:

```bash
scp ais_uzmo-1.0.0.tar.gz SHA256SUMS RELEASE.txt root@АДРЕС_СЕРВЕРА:/root/
```

Замените `АДРЕС_СЕРВЕРА` и номер версии на реальные значения.

## 3. Проверка полученного архива

Подключитесь к серверу по SSH. Если вошли не как `root`, сначала выполните
`sudo -i`. Все следующие команды выполняются от `root`.

Проверьте архив:

```bash
cd /root
sha256sum -c SHA256SUMS
```

Должно появиться:

```text
ais_uzmo-1.0.0.tar.gz: OK
```

Если вместо `OK` показана ошибка, остановитесь и запросите архив повторно.

## 4. Распаковка

Для версии `1.0.0` выполните:

```bash
mkdir /root/ais_uzmo-install-1.0.0
tar -xzf /root/ais_uzmo-1.0.0.tar.gz \
  -C /root/ais_uzmo-install-1.0.0 --strip-components=1
cd /root/ais_uzmo-install-1.0.0
```

Для другой версии замените `1.0.0` во всех трёх местах.

## 5. Заполнение четырёх параметров

Создайте файл настроек установки:

```bash
cp deploy/install.env.example deploy/install.env
nano deploy/install.env
```

Заполните только значения справа от знака `=`:

```dotenv
SERVER_NAME=10.20.30.40
TRUSTED_CIDR=10.20.30.0/24
ADMIN_USERNAME=admin
ADMIN_EMAIL=admin@example.local
```

- `SERVER_NAME` — IP-адрес или внутреннее имя сервера;
- `TRUSTED_CIDR` — адрес одного компьютера или доверенной внутренней сети в
  формате, который даст сетевой администратор; не указывайте `0.0.0.0/0`;
- `ADMIN_USERNAME` — имя первой учётной записи руководителя;
- `ADMIN_EMAIL` — служебная почта руководителя. Если почты нет, оставьте пусто.

Сохранение в `nano`: `Ctrl+O`, `Enter`, затем `Ctrl+X`.

Пароли в этот файл записывать не нужно. Установщик создаст внутренние пароли сам,
а пароль руководителя попросит ввести без отображения на экране.

Внизу файла оставьте `ENABLE_UFW=true`. Строку `SSH_PORT=22` тоже не меняйте,
если подключались обычной командой `ssh`. Если подключались с параметром вроде
`ssh -p 2222`, укажите тот же номер: `SSH_PORT=2222`.

## 6. Установка

Не закрывайте текущий SSH-сеанс до окончания проверки. Выполните:

```bash
bash deploy/install.sh deploy/install.env
```

Установка может занять несколько минут. Если команда завершилась с ошибкой, не
запускайте её много раз подряд: сохраните текст ошибки и передайте разработчику.

## 7. Проверка

После успешной установки выполните:

```bash
bash /srv/ais_uzmo/deploy/check.sh
```

В конце должно быть сообщение:

```text
Все проверки пройдены.
```

Затем откройте в браузере адрес, указанный в `SERVER_NAME`, и проверьте:

1. открывается страница входа;
2. руководитель входит с созданным паролем;
3. открываются основные разделы;
4. создаётся контрольная заявка;
5. к заявке загружается и затем открывается фотография.

После проверки удалите контрольную заявку и фотографию.

После этого перезагрузите сервер:

```bash
reboot
```

Подключитесь снова и ещё раз запустите проверку:

```bash
bash /srv/ais_uzmo/deploy/check.sh
```

## 8. Резервные копии

Резервная копия создаётся автоматически около 03:30, хранится на сервере 14 дней
и находится здесь:

```text
/var/backups/ais_uzmo/daily/
```

Проверка расписания:

```bash
systemctl list-timers ais-uzmo-backup.timer
```

Создать копию немедленно:

```bash
systemctl start ais-uzmo-backup.service
systemctl show -p Result --value ais-uzmo-backup.service
```

Команда должна вывести `success`. Во время создания копии программа ненадолго
останавливается. Копия включает базу, фотографии, настройки порогов панели и
закрытые настройки. Её нужно регулярно переносить на другой защищённый сервер
или носитель: копия на том же диске не спасёт при поломке сервера.

## 9. Обновление программы

Разработчик передаёт новый архив и новый `SHA256SUMS`. Скопируйте их в `/root`,
проверьте контрольную сумму так же, как в разделе 3, затем выполните:

```bash
bash /srv/ais_uzmo/deploy/update.sh /root/ais_uzmo-1.1.0.tar.gz
```

Замените номер версии на полученный. Скрипт сначала создаёт резервную копию,
затем обновляет программу и запускает проверку.

При установке из архивов команда `git pull` не используется.

## 10. Если программа не открывается

Выполните четыре команды и передайте их полный вывод разработчику:

```bash
bash /srv/ais_uzmo/deploy/check.sh
systemctl status ais_uzmo.service --no-pager
journalctl -u ais_uzmo.service -n 100 --no-pager
nginx -t
```

Не удаляйте `.env`, базу, каталоги `media` и `runtime` или резервные копии для
попытки «установить заново».

## 11. Важные ограничения

- Текущая простая установка использует HTTP и рассчитана только на закрытую
  доверенную сеть. Для другого контура ответственный за сеть и ИБ должен
  предоставить VPN или ведомственную защиту. Если обязателен именно HTTPS,
  заранее запросите версию комплекта, в которой TLS поддержан установкой,
  проверкой и обновлением; одиночная ручная правка Nginx будет затёрта обновлением.
- Не открывайте наружу порты `8000`, `5432` и `6379`.
- Не храните исходный код в публичном репозитории.
- Не помещайте `.env`, пароли, базу и фотографии в архив исходного кода.
- Не отправляйте комплект через обычную почту, мессенджер или публичное облако
  без разрешения ответственного за ИБ.
